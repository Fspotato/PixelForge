## 1. 專案概述

### 1.1 定位

AI Service Framework 是一套以 **Django REST Framework** 為核心的 AI 服務後端框架，旨在提供：

- **多供應商 AI 接入**（OpenAI、Anthropic、Google、Azure OpenAI）
- **完整的認證與帳號管理**（JWT、社群登入、Email 驗證）
- **金流整合**（ECPay 綠界、Stripe、NewebPay 藍新）
- **分佈式任務處理**（Celery + Redis）
- **事件驅動架構**（事件匯流排 + 觀察者模式）
- **容器化部署**（Docker Compose 三環境）

### 1.2 技術棧

| 層級 | 技術 |
|------|------|
| 語言 | Python 3.12+ |
| Web 框架 | Django 5 + Django REST Framework 3.16 |
| 認證 | SimpleJWT（Access 15min / Refresh 7days） |
| 資料庫 | PostgreSQL 17（pgvector 擴充） |
| 快取/訊息代理 | Redis 8 |
| 非同步任務 | Celery 5.6 + django-celery-beat |
| API 文件 | drf-spectacular（OpenAPI 3.0） |
| 套件管理 | uv |
| 容器化 | Docker Compose |
| ASGI 伺服器 | Daphne（支援 WebSocket） |

### 1.3 統計數據

| 指標 | 數值 |
|------|------|
| 核心模組 Python 檔案 | 91 個 |
| 核心模組程式碼行數 | ~4,150 行 |

## 2. 架構設計

### 2.1 分層架構

```
┌─────────────────────────────────────────────────┐
│                   API Layer                      │
│         (Views / Serializers / URLs)             │
├─────────────────────────────────────────────────┤
│               Service Layer                      │
│     (AccountService / PaymentService / ...)      │
├─────────────────────────────────────────────────┤
│                Model Layer                       │
│      (User / ProviderConfig / Transaction)       │
├─────────────────────────────────────────────────┤
│            Core Internal Modules                 │
│    (_common / _logger / _event_bus / _task_queue) │
├─────────────────────────────────────────────────┤
│              Infrastructure                      │
│     (PostgreSQL / Redis / Celery / Docker)        │
└─────────────────────────────────────────────────┘
```

### 2.2 模組依賴方向

```
外部模組 (accounts, auth, ai_providers, payments)
    ↓ 依賴
內部模組 (_common, _logger, _event_bus, _task_queue)
    ↓ 依賴
框架設定 (config/)
```

**原則**：
- 以 `_` 前綴標示的模組為內部模組，不對外暴露 API
- 外部模組之間避免直接依賴，透過事件匯流排解耦
- 所有 Model 的外鍵使用 `settings.AUTH_USER_MODEL` 字串避免循環引入

### 2.3 命名慣例

| 類型 | 慣例 | 範例 |
|------|------|------|
| 模組目錄 | snake_case | `ai_providers/` |
| 內部模組 | `_` 前綴 | `_event_bus/` |
| Model 類別 | PascalCase | `PaymentTransaction` |
| API 路徑 | kebab-case | `/api/v1/ai-providers/` |
| 設定常數 | UPPER_SNAKE | `CELERY_BROKER_URL` |

## 3. 模組總覽

### 3.1 `_common` — 共用基礎模組

**狀態**：✅ 已完成（先前實作）

提供所有模組共用的基礎類別：

- **BaseModel**：UUID 主鍵 + 時間戳 + 軟刪除
- **TimestampMixin / UUIDPrimaryKeyMixin / SoftDeleteMixin**：可組合的 Model Mixin
- **BaseSerializer / BaseViewSet**：統一的序列化與視圖基礎類別
- **CustomExceptionHandler**：全域例外處理，回傳統一格式
- **StandardPagination**：統一分頁設定
- **ServiceRegistry**：服務註冊中心

### 3.2 `_logger` — 日誌模組

**狀態**：✅ 已完成（先前實作）

結構化日誌系統：

- **RequestLoggingMiddleware**：自動記錄每個 HTTP 請求
- **RotatingFileHandler**：按日期輪替日誌檔案
- **JSON Formatter**：結構化 JSON 日誌格式
- 日誌位置：`backend/logs/<env>-YYYY-MM-DD.log`

### 3.3 `_event_bus` — 事件匯流排

**狀態**：✅ 本次實作  
**檔案數**：5 個  
**提交**：`68c4076`

輕量級觀察者模式事件系統：

- **EventEnvelope**：封裝事件名稱、資料、metadata
- **EventRegistry**：支援萬用字元（`*`、`?`）的事件訂閱
- **EventBus**：同步 + 非同步（Celery）雙模式派發
- **publish_event() / subscribe()**：頂層公開 API

```python
from core._event_bus import publish_event, subscribe

subscribe("user.created", handle_new_user)
publish_event("user.created", {"user_id": "..."})
```

### 3.4 `_task_queue` — 分佈式任務佇列

**狀態**：✅ 本次實作  
**檔案數**：7 個  
**提交**：`2504690`

基於 Celery 的任務佇列，提供進度追蹤與重試策略：

- **BaseTask**：可繼承的任務基礎類別，內建上下文管理（request_id, user_id）
- **TaskProgress Model**：資料庫層級的任務進度追蹤
- **RetryPolicies**：預定義重試策略（immediate / exponential / conservative）
- **Admin**：Django Admin 介面管理任務

### 3.5 `accounts` — 帳號管理

**狀態**：✅ 本次實作  
**檔案數**：11 個  
**提交**：`9818153`

以 Email 為主體的使用者模型：

- **User Model**：UUID 主鍵、Email 登入（無 username）、頭像、狀態管理
- **SocialAccount Model**：社群帳號綁定
- **EmailVerification Model**：Email 驗證令牌
- **AccountService**：帳號業務邏輯（更新資料、停用、換信）
- **API**：個人資料 CRUD、頭像上傳、帳號停用、社群帳號管理

### 3.6 `auth` — 認證模組

**狀態**：✅ 本次實作  
**檔案數**：15 個  
**提交**：`fb63c21`

完整的認證流程：

- **JWT 認證**：Access Token（15 分鐘）+ Refresh Token（7 天），支援黑名單
- **Email 認證後端**：支援 Email + 密碼登入
- **社群登入**：Google OAuth2、GitHub OAuth（可擴充）
- **速率限制**：登入 5 次/分、註冊 3 次/分、密碼重設 3 次/時
- **API 端點**：登入、登出、註冊、Email 驗證、密碼重設、社群登入

### 3.7 `ai_providers` — AI 供應商接入

**狀態**：✅ 本次實作  
**檔案數**：18 個  
**提交**：`a57211e`

多供應商 AI 服務接入層：

- **BaseProvider ABC**：供應商抽象介面
- **ProviderRegistry**：供應商註冊與查找
- **四個內建供應商**：OpenAI、Anthropic、Google Gemini、Azure OpenAI
- **AIProviderService**：統一呼叫介面，支援自動 fallback
- **API Key 加密**：使用 cryptography.Fernet，金鑰衍生自 SECRET_KEY
- **使用量追蹤**：UsageRecord 記錄每次 Token 使用
- **SSE 串流**：支援 Server-Sent Events 串流回應
- **Schemas**：ChatRequest / ChatResponse / EmbeddingRequest 等資料結構

### 3.8 `payments` — 金流接入

**狀態**：✅ 本次實作  
**檔案數**：15 個  
**提交**：`380482c`

金流閘道適配器模式：

- **BaseGateway ABC**：金流閘道抽象介面（create_order / verify_callback / refund / query）
- **GatewayRegistry**：閘道註冊與查找
- **三個內建閘道**：ECPay 綠界、Stripe、NewebPay 藍新
- **PaymentTransaction Model**：交易記錄（含狀態機）
- **PaymentLog Model**：操作日誌（狀態變更審計軌跡）
- **PaymentService**：統一支付流程（建立 → 回調驗證 → 退款）
- **Webhook 端點**：接收金流平台回調通知

## 5. 資料模型

### 5.1 模型關係圖

```
User (accounts.User)
 ├── 1:N → SocialAccount (accounts.SocialAccount)
 ├── 1:N → EmailVerification (accounts.EmailVerification)
 ├── 1:N → ProviderConfig (ai_providers.ProviderConfig)
 ├── 1:N → UsageRecord (ai_providers.UsageRecord)
 └── 1:N → PaymentTransaction (payments.PaymentTransaction)
                └── 1:N → PaymentLog (payments.PaymentLog)

TaskProgress (_task_queue.TaskProgress)  ← 獨立，無 FK
```

### 5.2 模型列表

| 模組 | Model | 主要欄位 |
|------|-------|---------|
| accounts | **User** | email（唯一）、first_name、last_name、avatar、status、settings_data |
| accounts | **SocialAccount** | user（FK）、provider、provider_uid、access_token、refresh_token |
| accounts | **EmailVerification** | user（FK）、token（唯一）、verified_at、expires_at |
| ai_providers | **ProviderConfig** | owner（FK）、provider_name、api_key_encrypted、default_model、fallback_provider |
| ai_providers | **UsageRecord** | user（FK）、provider_name、model、prompt_tokens、completion_tokens、is_fallback |
| payments | **PaymentTransaction** | user（FK）、gateway、amount、currency、status、paid_at、refunded_at |
| payments | **PaymentLog** | transaction（FK）、action、old_status、new_status、raw_data |
| _task_queue | **TaskProgress** | celery_task_id（唯一）、task_name、status、progress、result_data |

### 5.3 Mixin 基礎類別（`_common`）

| Mixin | 提供欄位 |
|-------|---------|
| **UUIDPrimaryKeyMixin** | `id`（UUID4 主鍵） |
| **TimestampMixin** | `created_at`、`updated_at`（自動管理） |
| **SoftDeleteMixin** | `deleted_at`、`is_deleted` |
| **BaseModel** | 以上三者組合 |
