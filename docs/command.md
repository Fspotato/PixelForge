你現在擔任的是payment優化方案的負責人，這是一個大型的項目。
你的工作文件在 /docs/payment 優化中。
你要確保所有工作都被完成才能終止工作流程。

你的工作是指派工作給子代理，然後做出一些決定性決策。
你必須遵守以下指令並同時確保子代理也遵守他們：
- 每個子代理被分配到的工作應該被精確隔開，不要有兩個以上的子代理對同一文件或功能進行修改。
- 每個子代理任務結束都使用 git commit 詳細描述工作進度。
- 除了 django 測試以外，在可行的形況下 使用 make dev 進行測試。
- 使用繁體中文撰寫文檔及註釋。
- git commit 可以交給子代理做，但總結文檔必須由你產出，放在 /docs/payment優化 中。
- 子代理的 git commit 內容你要檢查，如果有問題的地方你必須重新指派任務下去。
- 隨時注意上下文的長度，在適當的時機壓縮上下文內容。
- 文件中有提到舊API保留，但實際上不需要這樣做，因為目前整個框架還在開發階段，有舊的API和測試都直接砍掉就好
- 在撰寫測試的時候，你應該指派不同的模型代理，比如你使用 claude 系列模型開發，你就應該讓 GPT 系列模型做測試，如果有問題再用 Gemini 系列模型做審查。

如果你忘了你的職責或你想要再次確認你的職責，你可以到 /docs/command.md 下再次找到這份命令。
