# 前端測試面板優化報告

## 概述

針對前端 API 測試面板進行三大類修正與優化：修正測試案例預設格式錯誤、新增使用提示元件、全面 UI 體驗優化。

## 一、測試案例格式修正（7 項）

以下測試案例的預設 request body 與後端 serializer 實際接受的格式不一致，已全部修正：

| 測試案例 | 修正內容 |
|---------|---------|
| `notifications-preferences-update` | path 參數從 `{channel}` → `{category}`；body 欄位 `enabled` → `enabled_channels`（陣列），新增 `is_muted`、`quiet_hours_timezone` |
| `rbac-roles-create` | 補充必填欄位 `is_default`（布林）、`parent`（UUID 或 null） |
| `rbac-roles-permissions-set` | 欄位名 `permissions` → `permission_ids`（UUID 陣列） |
| `rbac-check-permission` | 欄位名 `permission`（字串）→ `permissions`（陣列），新增 `check_all`（布林） |
| `rbac-user-roles-assign` | 補充可選欄位 `expires_at`（ISO 日期或 null） |
| `api-keys-create` | 補充可選欄位 `description`、`allowed_ips`（IP 白名單陣列） |
| `files-presign` | 補充可選欄位 `folder`、`visibility`、`backend` |

## 二、使用提示元件（UsageHint）

### 需求

使用者反饋許多測試端點需要先知道 UUID 或完成前置操作才能測試，缺乏使用說明。

### 實作

- **新增 `UsageHint.tsx` 元件**：位於送出按鈕與回應面板之間
- **提示來源**：優先使用每個 `TestCase` 的 `hint` 欄位，若無則自動產生預設提示
- **視覺區分**：以 ⚠️ 開頭的提示顯示為紅色警告樣式，一般提示顯示為藍色資訊樣式
- **全部 50 個測試案例**皆已加入繁體中文使用提示，涵蓋：
  - 前置步驟（如「請先使用『列出所有角色』取得角色 ID」）
  - 注意事項（如「建立後會回傳完整金鑰，僅此一次顯示」）
  - 參數說明（如「check_all 為 true 時需全部權限皆符合」）

### 型別擴充

`TestCase` 介面新增可選欄位：

```typescript
/** 使用提示：說明如何使用此測試案例、前置步驟、注意事項等 */
hint?: string
```

## 三、UI 體驗優化

### 3.1 側邊欄搜尋（Sidebar）

- 認證狀態下方新增搜尋框
- 支援依**名稱**、**路徑**、**描述**三個維度模糊搜尋
- 無搜尋結果時顯示「無符合的端點」提示
- 50 個端點快速定位不再需要逐項捲動

### 3.2 鍵盤快捷鍵（RequestPanel）

- 新增 **Ctrl+Enter**（Mac: **⌘Enter**）快速送出請求
- 按鈕旁顯示快捷鍵提示文字
- 自動偵測作業系統以顯示對應的修飾鍵

### 3.3 JSON 語法驗證（RequestPanel）

- 即時驗證 request body 的 JSON 格式
- 格式錯誤時 textarea 邊框變為紅色
- 下方顯示「JSON 格式錯誤」提示訊息
- 防止因 JSON 語法錯誤導致不明確的 API 錯誤

### 3.4 Response Headers 顯示（ResponsePanel）

- 新增可收合的 **Response Headers** 區塊
- 顯示 header 數量計數
- 預設收合，需要時展開查看完整回應標頭
